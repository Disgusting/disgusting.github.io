UI Design through feedbacks

We showcased our game during an early stage of development, this has been helpful in designer UI/UX of the game.

Early UI/UX design prototype.

- Talk about skeumorphic/diegetic UI design. UI still in game-space. Knife-Cut to communicate that reward structure.

Some of the feedbacks were:

  • Being able to cut while holding knife.
  • Unclear what the objective or the rewards will be in term of stars.
  • Cutting needed to be more satisfying.
  • They liked how the game could be completed with any number of cuts.
  • Some of hte players struggle to understand the user interace.
  • There should be grid overlay to help player visualize grid size.
  • Some player struggle to understand what the knife does.

The feedbacks leads to interface design such as the Cursor being reactive-- it changes sprites based on which object it's currently hoving over and the current state it's on (isGrabbingPiece, isGrabbingKnife).
